GUÍA PARA NUEVOS COLABORADORES
------------------------------


1. Herramientas de Desarrollo

El programa está desarrollado en Python, de modo que funciona tanto en
Windows como Linux. Las bibliotecas necesarias para poder desarrollar el
programa son las mismas que se utilizan para ejecutarlo. Consultar la guía
de instalación.

A la hora de descargar el GTK runtime es preferible descargar la versión de
desarrollo
( http://gladewin32.sourceforge.net/modules/wfdownloads/visit.php?lid=99 ),
ya que incluye Glade, la herramienta de edición gráfica de interfaces de
usuario.

2. Sistema de revisión de código

El código fuente se guarda en un repositorio, un sistema de control de
versiones. Esto significa que muchas personas pueden trabajar a la vez sobre
el mismo proyecto sin interferirse, y puesto que cada cambio queda
almacenado en caso de haber introducido un error es trivial volver atrás a
una versión que sí funcionaba.

El sistema de control de versiones es SVN. En windows es posible acceder a
él directamente desde el explorador utilizando un pequeño programa
disponible en http://tortoisesvn.tigris.org/ .
Entramos en la seccion .Downloads. y nos bajamos el .Official version for Win2k/XP or higher.
De la página que nos muestre, seleccionamos el "Download TortoiseSVN-1.2.4.4479-svn-1.2.3.msi".
(O la última versión disponible) Nos vuelve a salir una nueva página (para
los entendidos se llaman MIRRORS). Seleccionamos uno cualquiera. Si no se
descarga nada, nos aparece el siguiente mensaje:
 .Your download should begin shortly. If it does not, try 
  http://optusnet.dl.sourceforge.net/sourceforge/tortoisesvn/TortoiseSVN-1.2.4
  .4479-svn-1.2.3.msi or choose a different mirror.
Pues eso, o hacemos clic sobre el enlace propuesto, o volvemos a seleccionar un Nuevo mirror.
Nos sale otra vez la opción de ejecutar o guardar (recomendamos ejecutar)

En Linux hay también
interfaces gráficos: el esvn está bastante bien, y el tksvn está un poco
verde pero tiene buena pinta. Hay también un módulo para el editor emacs,
que se llama psvn.

Para obtener la dirección del repositorio y un usuario y contraseña,
consultar directamente al crujimaster.

Hacer un checkout para obtener una copia de trabajo de la última
versión disponible. Para hacer un checkout, tenemos que hacer clic con el
botón derecho, y en la ventana que aparece seleccionar: SVN Checkout.
Se abre una ventana, y en el primer campo ponemos la url del repositorio. En
el segundo campo seleccionamos la carpeta donde queremos guardar todos los
archivos, incluido el de ejecutar el crujisim. Le damos a OK y se empieza a
descargar por primera vez el programa al completo en la carpeta deseada. 

Hacer updates regularmente para actualizar la copia de trabajo con los
cambios que hayan introducido el resto de compañeros. Para hacer Update,
pues parecido al Checkout. Te vas a la carpeta esa nueva que acabamos de
crear y donde hemos metido todos los archivos esos descargados con la
operación anterior. Hacemos clic con el boton derecho, y nos sale una
ventanita, y elegimos SVN UPDATE. El sistema se descargará automáticamente
las últimas modificaciones realizadas por los compis en el programa, así
como las últimas pasadas introducidas. Hacer un commit cada vez que nosotros
hagamos un cambio sobre un archivo para que los compañeros puedan verlo
cuando ellos hagan un commit. No importa si dos personas han hecho cambios
sobre un mismo archivo. El sistema integra ambos cambios. (Si casualmente
las dos personas han tocado en la misma parte de un archivo, el subversion
informará al segundo de que no puede integrar automáticamente los cambios,
para que lo haga él mismo a mano.)

Existe un librito en línea que explica con más detalle el funcionamiento del
sistema. Está disponible en español en 
http://svnbook.red-bean.com/es/1.1/index.html
